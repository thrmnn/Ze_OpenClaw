## Safety Protocols (ALL papers)
1. Pre-flight check: clean working tree, synced remote
2. Backup before any changes
3. User approval required for ALL content changes
4. Show diff before applying
5. Commit immediately after change
6. Push to GitHub after commit

## Constraints
- NEVER modify scientific claims without approval
- NEVER change citations/figures without approval
- ALWAYS read CLAUDE.md editing guidelines first
- ALWAYS use safety-protocols.sh workflow
